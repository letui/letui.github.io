再谈C之头文件和JAVA接口 # 编程语言 # 2018-09-27
头文件是扩展名为 .h 的文件， 包含了 C 函数声明和宏定义，被多个源文件中引用共享。 有两种类型的头文件：程序员编写的头文件和编译器自带的头文件。 以上就是比较详细的阐述了。 划重点：头文件中包含了函数声明和宏定义。 四个字：函数声明。 [returnType] [methodName]([paramsType][paramsName]...); 基本上就是 返回值类型，函数名称，括（形参类型：形参名称...）号+分号 那么，声明完了，要完成函数的定义吧。 函数的定义是啥呢?就是为上述的声明加上方法体{...} int add(int x,int y); int add(int x,int y){ return x+y; } 完整示例就是上面的这段代码了。从语法语义深层次上，上面的代码和下面的是一样的。 int a; a=12; 可以自己偏执的认为，方法就是一种特殊的变量,遵循着特殊的语法。 按照C语言的设计，头文件负责写函数声明，源文件负责写函数定义(实现)。 这样的规定就跟Java里:interface写函数声明，实现类里写函数实现一样一样滴。 也就是说 从某种层次上来说，C的头文件契合了Java里的interface。但是在C里同一个头文件里的函数声明 不允许有多个同样函数定义(实现)，哪怕这些函数实现在不同名称的源文件里。（因为C没有类的概念， 所以，C语言里的变量，函数，的唯一身份标识，就是他们的名称。） （划重点：以上说法至少是编译时不允许） 然而，Java允许啊，Java里的一个interface允许多个实现类啊。是啊，还不是拖有类(class)的福？ Java里的类，把接口里声明的函数实现了，这时候实现类里的函数的唯一标识和类名绑在了一起，自然就不会 引起编译错误。 所以呢？C里面要想实现JAVA里一个接口多个实现那种编程范式，只能利用函数声明+函数定义+函数指针。 到此为止吧，有机会把之前的C语言 实现继承的方式写出来。
